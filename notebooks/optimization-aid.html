Aiding a City after a Natural Disaster: An Optimization Problem

Imagine that you in charge of the Operations Research division of a humanitarian organization, specialized in providing a fast response in assisting the population after a natural disaster. Identify and describe an optimization problem that may contribute to improve this organization response (food logistics, infrastructure reconstruction, vaccination programs, etc.,), where the decision-making process requires the formulation of a (continuous) linear optimization problem (LP) with at least 100 decision variables and 20 constraints. You should not use any of the models described in class or in the slides (but a significant modification of any of them might be acceptable).

Our case:

Unfortunately, we have once again seen the effects of the floods in the Valencia region. The floods of October 2024 have caused enormous damage in many towns in the province of Valencia, but not in the capital. Why?

In 1957, Valencia experienced one of its worst floods, but it was the last one. As a result of this catastrophe, the course of the Turia, its river, was diverted, thus avoiding future floods, including this year's. Now, the people of the affected localities need this type of works to be able to continue living peacefully.

We are going to make a model that simulates what would have happened if the new Turia riverbed had never been created and we had to provide help to the 16 districts of Valencia.

a) Formulation of the problem

Formulate mathematically the problem as a linear optimization model (based on the general formulation of an LP). Identify the model sets, parameters, variables, objective function and constraints.

This model is going to use data that would be available before the natural disaster about the risk of each zone and data collected after the disaster about necessities. This allows us to make a more precise model in the time between natural disasters but also to take into account how it has affected the different zones.

Variables and parameters

Our data will mix real data from the city of Valencia and random data created based in the real world. This random data will always be created using a random.seed, so the dataset can be retrieved and the results reproduced.

  • $x_{ij}$: tons of supply $j$ sent to district $i$. These will be our decision variables.

  • $v_i$: vulnerability index of the district $i$ of Valencia in the year 2021. It supports users to assess a communitys vulnerability to climate hazards and impacts such as flooding, extreme heat and drought events. The mapped outputs can be used to inform decision-making on climate adaptation and resilience building. These values are calculated from https://valencia.opendatasoft.com/explore/dataset/vulnerabilidad-por-barrios/table/.

  • $p_{ij}$: priority index for supply $j$ in district $i$. It will assign a random number in a subrange between the range [0, 1]. This subrange will be assigned depending on the severity that the flooding of 1957 had in the different districts in Valencia: very low (0), low (1), medium (2), high (3). We can suppose what the severity would have been in the newer districts by checking their proximity to the river Turia. In a real case, a simple formulary sent to the district governors could provide us this information.

  • $S_j$: available tons of supply $j$.

  • $C_i$: capacity of storage for tons of supplies for district $i$.

  • $R_{ij}$: minimum tons of supply $j$ required by district $i$.

  • $t_i$: time of transport by ton of supply for district $i$.

  • $L_i$: limit of total time of transportation for district $i$.

Objective function

Our objective will be to maximize the following expression:

$$ \max \sum_{i=1}^m v_i^2 \sum_{j=1}^n p_{ij}x_{ij} $$

where we square the vulnerability index to increase its relevance. Maximizing this function will tend to assign more supplies to those districts that require it and those that are more vulnerable or have been the most affected.

Constraints

These constraints can be adapted to several types of real life situations, which makes our model very flexible.

  • Non-negativity of the decision variables: $ x_{ij} \in \mathbb{R}_{\ge 0} $

  • Limit of supplies: $ \sum_{i=1}^m x_{ij} \le S_j $. There is a total amount of supply $j$, so we cannot excede that limit.

  • Supply capacity: $ \sum_{j=1}^n x_{ij} \le C_i $. Each district has a limit of capacity for storing supplies.

  • Minimum supplies required: $ x_{ij} \ge R_{ij} $. Some districts have asked for a minimum amount of supplies that we have to provide.

  • Time limit: $ t_i \sum_{j=1}^n x_{ij} \le L_i $. Time taken to deliver a ton of supply and time limit varies between district, and we cannot excede it.

  • Linked supplies: $ x_{i3} = x_{i4} \ \rightarrow \ x_{i3} - x_{i4} = 0 $. Supplies 3 and 4 are transported always together, so tons transported have to be equal

  • Proportion of supply: $ x_{i6} \le 0.5 \sum_{j=1}^n x_{ij} \ \rightarrow \ x_{i6} - 0.5 \sum_{j=1}^n x_{ij} <= 0 $. Quantity of supply 6 cannot excede half of the quantity of total supplies in each district.

Create the model:

In[249]:
# Model
model = ConcreteModel()

# Sets
model.DISTRICTS = Set(initialize=range(1, m + 1))
model.SUPPLIES = Set(initialize=range(1, n + 1))

# Parameters
model.v = Param(model.DISTRICTS, initialize=v)
model.p = Param(model.DISTRICTS, model.SUPPLIES, initialize=p)

model.S = Param(model.SUPPLIES, initialize=S)
model.C = Param(model.DISTRICTS, initialize=C)
model.R = Param(model.DISTRICTS, model.SUPPLIES, initialize=R)
model.t = Param(model.DISTRICTS, initialize=t)
model.L = Param(model.DISTRICTS, initialize=L)

# Decision variables
model.x = Var(model.DISTRICTS, model.SUPPLIES, domain=NonNegativeReals)

Implement the objective function and the constraints:

In[250]:
# Objective function
def objective_rule(model):
    return sum((model.v[i])**2 * sum(model.p[i, j] * model.x[i, j] for j in model.SUPPLIES) for i in model.DISTRICTS)
model.OBJ = Objective(rule=objective_rule, sense=maximize)
In[251]:
# Constraint 1: Supply availability (of the supply j)
def supply_availability_rule(model, j):
    return sum(model.x[i, j] for i in model.DISTRICTS) <= model.S[j]
model.SupplyConstraint = Constraint(model.SUPPLIES, rule=supply_availability_rule)

# Constraint 2: Zone capacity for supplies (for the zone i)
def zone_capacity_rule(model, i):
    return sum(model.x[i, j] for j in model.SUPPLIES) <= model.C[i]
model.CapacityConstraint = Constraint(model.DISTRICTS, rule=zone_capacity_rule)

# Constraint 3: Minimum required supply (for the zone i and the supply j)
def min_required_supply_rule(model, i, j):
    return model.x[i, j] >= model.R[i, j]
model.MinRequiredSupplyConstraint = Constraint(model.DISTRICTS, model.SUPPLIES, rule=min_required_supply_rule)

# Constraint 4: Limit of time (for zone i)
def time_limit_rule(model, i):
    return model.t[i] * sum(model.x[i, j] for j in model.SUPPLIES) <= model.L[i]
model.TimeConstraint = Constraint(model.DISTRICTS, rule=time_limit_rule)

# Constraint 5: Linked supplies (supplies 3 and 4, for each zone i)
def linked_supplies_rule(model, i):
    return model.x[i, 3] == model.x[i, 4]
model.LinkedSupplies = Constraint(model.DISTRICTS, rule=linked_supplies_rule)

# Constraint 6: Proportion of supply (supply 6, for each zone i)
def proportion_supply_rule(model, i):
    return model.x[i, 6] <= 0.5 * (sum(model.x[i, j] for j in model.SUPPLIES))
model.ProportionSupply = Constraint(model.DISTRICTS, rule=proportion_supply_rule)

d) Dual model

Formulate the dual associated with a), and implement and solve it in Pyomo. Does the Strong Duality Theorem holds? justify your answer.

The variables associated with each of the original constraints will be:

  • Limit of supplies: $ \sum_{i=1}^m x_{ij} \le S_j \ \rightarrow \ u_j \ge 0 $

  • Supply capacity: $ \sum_{j=1}^n x_{ij} \le C_i \ \rightarrow \ w_i \ge 0 $

  • Minimum supplies required: $ x_{ij} \ge R_{ij} \ \rightarrow \ y_{ij} \ge 0 $

  • Time limit: $ t_i \sum_{j=1}^n x_{ij} \le L_i \ \rightarrow \ z_i \ge 0 $

  • Linked supplies: $ x_{i3} - x_{i4} = 0 \ \rightarrow \ \mu_i \text{ free} $

  • Proportion of supply: $ x_{i6} - 0.5 \sum_{j=1}^n x_{ij} <= 0 \ \rightarrow \nu _i \ge 0 $

The objective function for our dual model is then:

$$ \min \sum_{j=1}^n u_j S_j + \sum_{i=1}^m w_i C_i + \sum_{i=1}^m z_i L_i - \sum_{i=1}^m \sum_{j=1}^n y_{ij} R_{ij} $$

And the constraint is:

$$ u_j + w_i + t_i z_i - y_{ij} + \alpha_i + \beta_i \ge v_i^2 p_{ij} $$

where

$$ \alpha_i = \begin{cases} \mu_i \ ; \ j = 3 \\ - \mu_i \ ; \ j = 4 \\ 0 \ ; \ j \ne \{3, 4\} \end{cases} $$

and

$$ \beta_i = \begin{cases} \nu_i \ ; \ j = 6 \\ - \nu_i \ ; \ j \ne 6 \end{cases} $$

e) Binary problem

Modify the problem in a) to impose some logical or conditional (linear) constraints that require the use of binary variables Implement and solve this new model in Pyomo and interpret the results. Include the data used for the solution of the problem.

We are going to create a problem similar to the one that we have solved, but in this case we want to know whether to send supplies $j$ to district $i$ or not, so we will use binary decision variables. We will also create some different constraints so we can explore their effects.

Formulation of the problem

Objective function:

It will be the same one as before,

$$ \text{max} \sum_{i=1}^m v_i^2 \sum_{j=1}^n p_{ij}x_{ij} $$

, but this time $ x_{ij} \in \{ 0, 1 \} $, where $ x_{ij} = 1 $ if supply $j$ has to be sent to district $i$ and $ x_{ij} = 0 $ if not.

Constraints:

  • Limit of supplies: $ \sum_{i=1}^m x_{ij} \le S_j \ \ , \ \ S_j \in [10, 16] $. There is a limit of districts that we can send the supply $j$ to.

  • Costs limit: $ \sum_{i=1}^m \sum_{j=1}^n c_{ij} x_{ij} \le B = 20 $. Taking supply $j$ to district $i$ takes a certain cost $c_{ij}$ (for example, time), and there is a total limit $B$. This can be adapted to several type of real life constraints.

  • Linked supplies: $ x_{i3} = x_{i4} $. Supplies 3 and 4 are again linked together.

  • Required supplies: $ x_{ij} \ge R_{ij} $. There is a small probability (10%) that a district $i$ must need the supply $j$.

f) Problem size vs. Computational time

For the integer problem in e), solve several instances with dierent sizes (dierent numbers of variables and/or constraints) using randomly generated data. Graph the results (problem size vs computational time), and interpret them.

We are going to scale the problem in e) so it has more districts and more supplies. This can easily happen in the case where we have bigger cities than Valencia or when we have details of the necessary supplies for zones smaller than districts, so we can have more precission to help those who need it the most.

To scale it, we are going to add the parameter $i$ to $m$ and $n$ and solve the binary problem for this $m, n$. This means that the quantity of decision variables will be given by $ (19 + i) (6 + i) = i^2 + 25i + 114 $.

def binary_problem_variable_size(i):

    while True:

        time1 = time.time()

        #random.seed(42)

        m = 19 + i
        n = 6 + i

        # Generate random data
        v = {i: random.uniform(0, 1) for i in range(1, m + 1)}  # Index of vulnerability
        p = {(i, j): random.uniform(0, 1) for i in range(1, m + 1) for j in range(1, n + 1)}  # Priority

        S = {j: random.randint(1, m) for j in range(1, n + 1)}
        c = {(i, j): random.uniform(0, 1) for i in range(1, m + 1) for j in range(1, n + 1)}
        B = 20
        R = {(i, j): random.choices([0, 0], [0.9, 0.1])[0] for i in range(1, m + 1) for j in range(1, n + 1)}

        # Model
        model = ConcreteModel()

        # Sets
        model.DISTRICTS = Set(initialize=range(1, m + 1))
        model.SUPPLIES = Set(initialize=range(1, n + 1))

        # Parameters
        model.v = Param(model.DISTRICTS, initialize=v)
        model.p = Param(model.DISTRICTS, model.SUPPLIES, initialize=p)

        model.S = Param(model.SUPPLIES, initialize=S)
        model.c = Param(model.DISTRICTS, model.SUPPLIES, initialize=c)
        model.R = Param(model.DISTRICTS, model.SUPPLIES, initialize=R)

        # Decision variables
        model.x = Var(model.DISTRICTS, model.SUPPLIES, domain=Binary)

        # Objective function
        def objective_rule(model):
            return sum(
                (model.v[i])**2 * sum(model.p[i, j] * model.x[i, j] for j in model.SUPPLIES)
                for i in model.DISTRICTS
            )
        model.OBJ = Objective(rule=objective_rule, sense=maximize)

        #Constraints
        def constraint1(model, j):
            return sum(model.x[i, j] for i in model.DISTRICTS) <= model.S[j]
        model.Constraint1 = Constraint(model.SUPPLIES, rule=constraint1)

        def constraint2(model):
            return sum(sum(model.c[i, j] * model.x[i, j] for j in model.SUPPLIES) for i in model.DISTRICTS) <= B
        model.Constraint2 = Constraint(rule=constraint2)

        def constraint3(model, i, j):
            return model.x[i, j] >= model.R[i, j]
        model.Constraint3 = Constraint(model.DISTRICTS, model.SUPPLIES, rule=constraint3)

        def constraint4(model, i):
            return model.x[i, 3] == model.x[i, 4]
        model.Constraint4 = Constraint(model.DISTRICTS, rule=constraint4)

        # Solve
        solver = SolverFactory('glpk')
        results = solver.solve(model)

        # If it has not found an optimal solution, run again:
        if results.solver.status == SolverStatus.ok and results.solver.termination_condition == TerminationCondition.optimal:
            break
        else:
            continue
        
    time2 = time.time()

    return time2-time1

In our constraints for the binary problem, we have a costs limit of the form: $ \sum_{i=1}^m \sum_{j=1}^n c_{ij} x_{ij} \le B = 20 $. As this constraint is very interesting because many real life constraints can be expressed with this sum of weights, we are going to see how the values of the objective function change when we change this parameter $B$.

In[279]:
def binary_problem_variable_parameter(B):

    random.seed(42)  # As we use the random seed, all data except B is going to be the same.

    m = 19
    n = 6

    # Generate data
    v = {i: random.uniform(0, 1) for i in range(1, m + 1)}  # Index of vulnerability
    p = {(i, j): random.uniform(0, 1) for i in range(1, m + 1) for j in range(1, n + 1)}  # Priority

    S = {j: random.randint(1, m) for j in range(1, n + 1)}
    c = {(i, j): random.uniform(0, 1) for i in range(1, m + 1) for j in range(1, n + 1)}
    # B = 20
    R = {(i, j): random.choices([0, 0], [0.9, 0.1])[0] for i in range(1, m + 1) for j in range(1, n + 1)}

    # Model
    model = ConcreteModel()

    # Sets
    model.DISTRICTS = Set(initialize=range(1, m + 1))
    model.SUPPLIES = Set(initialize=range(1, n + 1))

    # Parameters
    model.v = Param(model.DISTRICTS, initialize=v)
    model.p = Param(model.DISTRICTS, model.SUPPLIES, initialize=p)

    model.S = Param(model.SUPPLIES, initialize=S)
    model.c = Param(model.DISTRICTS, model.SUPPLIES, initialize=c)
    model.R = Param(model.DISTRICTS, model.SUPPLIES, initialize=R)

    # Decision variables
    model.x = Var(model.DISTRICTS, model.SUPPLIES, domain=Binary)

    # Objective function
    def objective_rule(model):
        return sum(
            (model.v[i])**2 * sum(model.p[i, j] * model.x[i, j] for j in model.SUPPLIES)
            for i in model.DISTRICTS
        )
    model.OBJ = Objective(rule=objective_rule, sense=maximize)

    #Constraints
    def constraint1(model, j):
        return sum(model.x[i, j] for i in model.DISTRICTS) <= model.S[j]
    model.Constraint1 = Constraint(model.SUPPLIES, rule=constraint1)

    def constraint2(model):
        return sum(sum(model.c[i, j] * model.x[i, j] for j in model.SUPPLIES) for i in model.DISTRICTS) <= B
    model.Constraint2 = Constraint(rule=constraint2)

    def constraint3(model, i, j):
        return model.x[i, j] >= model.R[i, j]
    model.Constraint3 = Constraint(model.DISTRICTS, model.SUPPLIES, rule=constraint3)

    def constraint4(model, i):
        return model.x[i, 3] == model.x[i, 4]
    model.Constraint4 = Constraint(model.DISTRICTS, rule=constraint4)

    # Solve
    solver = SolverFactory('glpk')
    solver.solve(model)
        
    return value(model.OBJ)

This means that increasing this limit will not give benefits (or they will be marginal) after this point, as the constraint is no longer relevant.

However, in real life increasing this parameter has a cost, but there is a point of maximum curvature from which we start getting deminishing returns. To automate the search for this point, which could be very useful for knowing up to which point we need to increase the limit, we can use the elbow method. For this, we could use a specialized python library like $\textsf{kneed}$ but, to avoid compatibility issues, we are going to plot a simple approximation to the elbow point that maximizes the distance to a line from the first to the last point:
